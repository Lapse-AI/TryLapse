## Executive summary

**Overall: 7.0 / 10** — This is a strong Phase 2 **prototype** that maps remarkably well to the monitoring platform design doc. IA, command center density, run detail matrix, agent roster, trends, and sitemap explorer are all present with credible demo data.

The biggest gap is **trust UX** — the core CEO mandate for Launch Rehearsal. Showing **Green** readiness alongside a **P0 SSO blocker**, an evidence button that does not surface artifacts, and inconsistent blocker counts would undermine founder confidence faster than missing features.

**Verdict:** Ship-ready as a **design partner demo** after fixing P0 trust issues (Batch A). Not ready for external marketing until interaction stubs and mobile layout are addressed (Batch B/C).

## What works well

### 1. Command center tells the right story immediately
- Headline *"Pre-launch readiness, observed."* + subcopy *"Evidence-bound, no auto-fix"* — perfect positioning vs competitors.
- Primary metrics visible without scroll: readiness, blockers, delights, duration, crawl size.
- Secondary KPI row (time-to-scorecard, flake rate, recurring blockers, agent cost) matches §8.2 spec.
- **Top blocker + top delight** side-by-side — implements CEO "not bugs-only" mandate.

### 2. Run detail is the product
- Interactive-grade **persona × journey matrix** (4×6) with journey metadata in column headers.
- Issues list includes severity, dimension, confidence, owner heuristic, recurrence, step_id (`j2.s4`).
- **Delights — required section** with voice-of-user quotes.
- Agent participation footer links back to control center — good cross-navigation.

### 3. Agent control center differentiates
- Collaboration trace (Crawler → … → Synthesizer) with stage status.
- Per-agent duration + cost — enterprise buyers will ask for this.
- Phase 2 agents (Compliance, Performance) shown as **idle** with explanation — honest scoping.
- Human-in-the-loop annotations block — maps to false-positive tracking metric.

### 4. Trends page is actionable
- Readiness avg vs goal, issue opened/resolved net, flake rate.
- Recurrence table with status (`open`, `new`, `regression`, `won't-fix?`) — exactly §8.6.
- Scheduled runs calendar stub with cron expression.

### 5. Site map explorer has the right data model
- Hub / leaf / orphan / auth taxonomy in list view.
- Graph view present (even if static).

## Issues (by severity)

### P0 — Trust-breaking (fix before any design partner)

| ID | Issue | Evidence | Fix |
|----|-------|----------|-----|
| **DR-01** | **Readiness Green with active P0 blocker** | Home shows `82 Readiness Green` + P0 SSO issue | Map score → semantic band: any P0 → **Red**; P1 → Amber max. Never Green with P0. |
| **DR-02** | **Blocker count mismatch** | Home: `Blockers 1`; run detail: 1× P0 + 3× P1 | Define blocker = P0+P1 (or P0 only) and use consistently everywhere. |
| **DR-03** | **Evidence button is a no-op** | Click `evidence` on run detail — no dialog, drawer, or navigation | Open evidence panel: screenshot + step_id + repro notes. This is the core product promise. |

### P1 — High impact UX

| ID | Issue | Evidence | Fix |
|----|-------|----------|-----|
| **DR-04** | **Matrix cells not interactive** | Cells are text (`fail`/`warn`/`pass`), not buttons/links | Make cells clickable → step timeline for that persona×journey. Add `aria-label="Prospect Priya, Pricing journey: fail"`. |
| **DR-05** | **Severity filter chips don't filter** | P0 click — full issue list still present in tree | Toggle filter state; hide non-matching severities; show active chip style + count. |
| **DR-06** | **Raw internal doc name in UI** | `EVALUATION_FRAMEWORK.md` rendered as label | Replace with user-facing "Evaluation dimensions" + optional tooltip linking to rubric. |
| **DR-07** | **Mobile layout broken** | At 390×844, full sidebar + all nav links still exposed | Collapse to hamburger sheet; pin readiness + Run CTA in sticky header. |
| **DR-08** | **Environment selector not interactive** | Breadcrumb text `acme / prod-canary / app.acme.io` only | Make env segment a dropdown per §8.2.5 (staging / prod-canary / demo). |

### P2 — Polish & consistency

| ID | Issue | Evidence | Fix |
|----|-------|----------|-----|
| **DR-09** | **Inconsistent button casing** | `crawl only`, `diff`, `export`, `replay`, `findings`, `configure` vs `Run rehearsal` | Sentence case for secondary; Title Case for primary CTA. |
| **DR-10** | **Idle agents show replay/findings** | Compliance + Performance idle but buttons enabled | Disable actions when idle; or hide buttons until enabled. |
| **DR-11** | **Site map graph is static image** | `img` with path list, not clickable nodes | SVG/canvas nodes with click → page preview panel (§8.4.3). |
| **DR-12** | **Missing Skeptic Sam persona agent** | Matrix has 4 personas; roster has 3 persona agents | Add 4th persona agent or document why matrix persona lacks agent lens. |
| **DR-13** | **Console error issue missing heading level in tree** | `'heading "Console error: ResizeObserver loop limit"'` malformed | Fix heading markup for a11y. |
| **DR-14** | **No active nav indicator** | All nav links identical in ARIA | `aria-current="page"` on active route. |
| **DR-15** | **Search ⌘K hint without command palette** | ⌘K text visible; not tested opening palette | Wire command palette or remove hint until implemented. |

### P3 — Nice-to-have

| ID | Issue | Fix |
|----|-------|-----|
| **DR-16** | Lovable "Edit with" badge overlaps content | Dismiss persists per session; hide in production builds |
| **DR-17** | Readiness shown as both `82` and `Green` without legend | Tooltip explaining numeric score vs band |
| **DR-18** | `warn` vs `partial` terminology | Align with scorecard DSL: pass / partial / fail |
| **DR-19** | Recent runs table lacks readiness color band | Color-code readiness column cells |
| **DR-20** | Export / diff buttons without menus | Add dropdown: PDF / Markdown / JSON |
