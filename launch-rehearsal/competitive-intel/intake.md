# Intake: Launch Rehearsal / TryLapse
*Skill: startup-competitors | Generated: 2026-06-15*

## Product
**TryLapse / Launch Rehearsal** — automated pre-launch behavioral testing of staging URLs. Submits a staging URL, simulates real user journeys with multiple persona types (power user, new user, mobile-first, enterprise cautious), scores the experience across 8 dimensions (Functionality, UI/UX, Information Clarity, Performance, Accessibility, Trust, Onboarding, Recovery), and produces an issue list with AI-driven suggestions and cross-run recurrence tracking.

## Problem & Customer
B2B SaaS teams who want to catch user-facing behavioral bugs and UX gaps before launch — without hiring a dedicated QA team or writing test scripts. Key pain: teams ship, get negative user feedback, then retrospectively understand what broke. Launch Rehearsal runs BEFORE launch (on staging) and gives a behavioral readiness score.

## Market Category
Pre-launch QA automation / AI-powered browser testing / UX quality scoring.

## Key Differentiators
1. **Staging-URL-first** — not a CI test runner, not a code-review tool; explicitly targets the pre-launch staging environment
2. **Persona-driven scoring** — tests as multiple distinct user archetypes, not generic Selenium scripts
3. **8-dimension readiness score** — quantified, calibrated, cross-run trend tracking
4. **Issue recurrence tracking** — marks issues as "new" vs "recurring across N runs" (longitudinal signal)
5. **Zero test authoring** — no scripts, no selectors, runs from a YAML config
