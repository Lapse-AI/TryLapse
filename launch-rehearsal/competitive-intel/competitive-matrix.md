## Market Gaps (Unserved by Any Competitor)

1. **Pre-launch behavioral readiness score** — nobody does this. All competitors frame testing as "developer CI/CD workflow," not "launch decision tool."
2. **Multi-persona behavioral comparison** — no competitor simulates multiple user archetypes and compares their experiences.
3. **Cross-run issue recurrence tracking** — nobody tracks "this issue appeared in 4 of 6 runs" as a feature.
4. **Industry benchmark comparison** — no competitor tells you how your score compares to similar products.
5. **8-dimension UX scoring framework** — no competitor has a named multi-dimensional readiness framework.

## Notes

- Greptile T-rex is a code-review-time unit test runner (pre-merge), not a staging URL tester (pre-launch) — categorically different
- Reflect's G2 profile is "inactive — not managed for over a year" — weak competitor
- QA.tech 13-month funding silence suggests fragility
- BrowserStack is the long-term platform risk, not a current direct competitor

## Red Flags

1. Launch Rehearsal is missing CI/CD integration — this means it currently requires explicit user action before each launch rather than running automatically
2. No G2 presence = no social proof for enterprise evaluation
3. No mobile testing = blocked from certain buyer segments

## Yellow Flags

1. Cross-browser coverage gap — worth noting in sales conversations but not a blocker for the target buyer (staging URL behavioral scoring doesn't require multi-browser at first)
2. Enterprise reference customers still needed (Phase 2 gate)
